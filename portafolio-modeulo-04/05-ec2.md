## Nombre stack:
- infra-viva-ec2

## ¿Qué es este componente?
Amazon EC2 (Elastic Compute Cloud) es el servicio de computación virtual de AWS. Para **Soluciones Digitales ACME**, la instancia EC2 actúa como el **servidor central de la aplicación**: procesa las solicitudes de los usuarios, se conecta a las bases de datos (RDS y DynamoDB) e interactúa con S3 para gestionar archivos.

## Instancia creada:
| Parámetro          | Valor                      |
| ------------------ | -------------------------- |
| Nombre             | VivaServer                 |
| Tipo de instancia  | t2.micro (1 vCPU, 1 GB RAM)|
| AMI                | Amazon Linux 2 (us-east-1) |
| Ubicación          | PublicSubnet (subred pública) |
| IP pública         | Asignada automáticamente   |
| Key Pair           | viva-key                   |

## Rol de EC2 dentro de la arquitectura ACME:
```
Usuario (internet)
       │
       ▼ (puerto 80 HTTP)
  VivaServer (EC2)
       │
       ├──→ RDS MySQL       (datos de pedidos, usuarios)
       ├──→ DynamoDB        (sesiones activas, datos en tiempo real)
       ├──→ S3              (descarga/subida de archivos)
       └──→ SNS/SQS         (notificaciones y mensajería)
```

## Crearemos lo siguiente:
| Recurso          | Detalle                                          |
| ---------------- | ------------------------------------------------ |
| Security Group   | Permite SSH (22) y HTTP (80)                     |
| Instancia EC2    | t2.micro con IP pública en subred pública        |
| Key Pair         | viva-key (para acceso SSH)                       |

## Prerequisito: crear Key Pair antes del stack
El Key Pair permite conectarse al servidor vía SSH. Debe crearse **antes** de lanzar el stack:

1. Ir a **EC2 → Key Pairs** en la consola AWS
2. Click → **Create key pair**
3. Nombre: `viva-key`
4. Formato: `.pem` (para Linux/Mac) o `.ppk` (para Windows con PuTTY)
5. Click → **Create** → el archivo se descarga automáticamente
6. Guardar el archivo en un lugar seguro (no se puede descargar de nuevo)

## Prerequisito: obtener IDs de red
Al desplegar el stack, reemplazar en `05-ec2.yml`:
```yaml
VpcId: vpc-XXXXXXXX       ← tu VPC ID (del stack 01-network)
SubnetId: subnet-XXXXXXXX ← ID de PublicSubnetA o PublicSubnetB
```

## Conectarse al servidor (SSH):
```bash
chmod 400 viva-key.pem
ssh -i "viva-key.pem" ec2-user@<IP_PUBLICA_EC2>
```

## Verificar
- Ir a **Amazon EC2**
- Entrar a: **Instances**
- Deberías ver: **VivaServer** con estado **Running** ✔
- Copiar la **Public IPv4 address**
- Abrir en el navegador: `http://<IP_PUBLICA>` para verificar acceso HTTP

> ⚠️ En producción se recomienda usar HTTPS (puerto 443) con un certificado SSL y restringir el SSH solo a IPs autorizadas en lugar de 0.0.0.0/0.
